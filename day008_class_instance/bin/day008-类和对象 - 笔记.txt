4. 构造方法：【重点掌握】Constructor
	4.1. 概念：就是一个类中的新结构，是类的成员。
	
	4.2. 构造方法特点：
		4.1	每一个类中都至少有一个构造方法;
		4.2	如果没有看到（显示的），那么存在一个隐式的无参数的构造方法;
		4.3	如果一个类中有显示的构造方法，那么隐式的无参的构造方法就不存在了;
	
	4.3. 作用：
		在创建对象的时候，直接给对象成员变量赋值，可以简化代码
		
		注意：只能在创建的时候赋值，不能给已经创建的对象赋值
		
		构造方法一旦通过 new调用，都表示在堆中创建新对象
		只要看得new，都表示在堆中创建新对象
		
	4.4. 使用：
		3.1.声明语法：【掌握】
			1.	构造方法的名字和类名完全一致（包括大小写完全一致）	
			2.	没有返回值类型（void也不行）	 
			3. 	构造方法方法体不能返回任何的值（即不能在方法体中return 值）
			4. 	其他的和普通方法类似 ，可以有修饰符（public、protected、private、默认）,
				可以有形参列表
			5.	构造方法必须有方法体
			6.	构造方法不能有任何非访问性质的修饰符修饰，例如 static、final、
				synchronized、abstract等都不能修饰构造方法。后面说
			
			语法：
				修饰符 Xxx(...){//Xxx业务类名，   ...是形参列表
					构造方法的方法体
				}
				
			设计类：
				public class Xxx{
					成员变量;
					
					修饰符 Xxx(...){
						构造方法的方法体
					}
					
					非static方法;
				}
			
			
		3.2.使用语法：【测试类中创建对象】
			语法：
				new 类名();// 调用无参构造
				new 类名(...);// 调用有参构造
			例如：	
				new User();// 表示在调用User的无参构造
	
	4.5. 注意事项：
		1.  普通方法不要和类名一致，行业默认规则;
		2.	普通方法（Method）和构造方法(Constructor)是不同的两种结构
		3.	构造方法调用方式：
				new 类名(...);
				
		4.	普通方法调用方式：
				看是否有static
				1. 有static，类名.方法名(...);
				2. 没有static，对象名.方法名(...);
		
		5. 只要是new 都表示在堆中创建一个新的对象
		
	4.6. 小结：【掌握】
		给对象成员变量赋值方式：
			1. 对象名.成员变量 = 值;// 值和成员变量数据类型一致
			2. 有参构造: new 类名(...);// 实参和成员变量数据类型一致
			
		对象取值方式：
			对象名.成员变量
		
		有参构造不能给已经创建的对象赋值，如果必须给已经创建的对象重新赋值，只能通过：对象名.成员变量 = 值;
		的方式实现
		
	4.7. 有参构造和无参构造使用场景：【重点掌握】
		当对象需要赋值的成员变量  很多 的时候，用：无参构造创建对象，然后用 对象名.成员变量 = 值;  方式赋值
		当对象需要赋值的成员变量  少 的时候，用：有参构造创建对象直接赋值即可

5. 匿名对象：（了解）
 		就是创建对象的时候，没有使用变量保存其堆中的地址。
 		例如：new _03User();//这就是一个匿名对象
 
 	使用场景：
 		只能在创建的时候直接使用，并且只能使用1次。
 
	作用：
 		可以节约资源，因为GC【垃圾回收器】会不定时回收匿名对象。
